1. Project Brief
Goal: rebuild bloodnexusstudio.in as a bold, motion-heavy marketing site using the structural, typographic, and animation system of apechain.com — adapted to BloodNexus branding. Content (games, services, team, careers, copy) comes from the existing BloodNexus site; this document defines structure, look, and behavior.
Who BloodNexus is (for design context): a Thane-based game development studio building immersive games with narrative depth, powered by Unreal Engine and Unity — cinematics, systems, worlds, plus VR and real-time 3D work. The site must feel like a AAA game studio: dark, cinematic, confident — which maps perfectly onto the reference site's 'giant type + cards + motion' formula.
Key adaptation decision: the reference site is electric-blue + cream (playful Web3). BloodNexus should keep the exact same layout, type scale, and animation system but flip the palette to a dark cinematic scheme — near-black canvas, blood-red primary, bone-white type. Same skeleton, different skin. Two palette options are given in §3; Option A is recommended.
Content note: the current site is a client-rendered React app, so copy could not be extracted automatically for this document. Every content slot below is marked [CONTENT: …] — pull the final text/images from the current site or the content sheet Anky provides. Structure and behavior in this doc are final.

8. Motion & Interaction System (identical to reference)
Global rules: default ease cubic-bezier(.22,1,.36,1); hovers 0.3–0.4s; reveals 0.6–0.8s, rise 24–40px + fade, trigger at 'top 85%', play once; stagger 60–100ms; ALL animations gated behind prefers-reduced-motion (fall back to opacity-only).
8.1 Text-roll link hover (use on every nav link, footer link, and button label)

  <span>View Game</span><span aria-hidden="true">View Game</span>
</span></a>

  transition:transform .35s var(--ease)}

8.3 Smooth scroll + reveals + parallax (GSAP)
const lenis=new Lenis({duration:1.1});
function raf(t){lenis.raf(t);requestAnimationFrame(raf)}
requestAnimationFrame(raf);lenis.on('scroll',ScrollTrigger.update);
 
gsap.utils.toArray('[data-reveal]').forEach(el=>{
  gsap.from(el,{y:32,opacity:0,duration:.7,ease:'power3.out',
    scrollTrigger:{trigger:el,start:'top 85%',once:true}})});
 
gsap.to('.hero__bg',{yPercent:12,ease:'none',
  scrollTrigger:{trigger:'.hero',start:'top top',
    end:'bottom top',scrub:true}});
8.4 Page transitions
Route change: black panel with thin red edge wipes up (0.5s expo.inOut) → route swap → panel wipes off → new hero plays load animation. Total < 0.9s. Implement with template.tsx + Framer Motion AnimatePresence or a GSAP timeline.
8.5 Card & media micro-interactions
Card hover: img scale 1 → 1.06 (0.5s), card translateY(-4px), red glow shadow.
Game cards: on hover, if a preview clip exists, crossfade key art → muted looping clip (load clip lazily on first hover).
Images fade in + scale 1.04 → 1 on lazy-load; reserve aspect-ratio boxes (CLS < 0.05).
Optional custom cursor dot (desktop only) that scales over interactive elements; becomes 'DRAG' label over the carousel.

10. Acceptance Checklist (definition of done)
Pixel behavior: text-roll on ALL links/buttons; marquee seamless (no jump); hero masked reveal on every page load; carousel drag + snap + peek; grid stagger reveals; page transition wipe.
Responsive: 375 / 768 / 1024 / 1440 breakpoints clean; hero uses 100svh; parallax + cursor disabled < 1024px.
Performance: Lighthouse ≥ 90 all categories; LCP < 2.5s on 4G; only transform/opacity animated; fonts preloaded woff2.
Accessibility: prefers-reduced-motion respected; duplicate roll/marquee text aria-hidden; :focus-visible states mirror hover; alt text on all art.
SEO (critical — current CRA site is invisible to crawlers): SSR/SSG all pages, per-page metadata + OG images, sitemap.xml, structured data (Organization + VideoGame schema on game pages).
CMS: games + careers editable without a deploy.
